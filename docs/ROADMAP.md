# Roadmap

## V1

- Python repository scan
- `requirements.txt` support
- `pyproject.toml` support
- source import scan via `ast`
- installed metadata fallback
- deterministic markdown, JSON, and receipt outputs
- exposure and authority classification for common runtime-expanding packages

## Near-Term

- richer local-module filtering
- better transitive graph coverage from installed metadata
- per-file import provenance in reports
- more precise build-only versus runtime separation
- test fixtures for Django, Celery, browser automation, and cloud SDKs

## V2

- generation-aware dependency governance for coding agents
- dependency introduction diffs across agent runs
- intended-versus-observed authority policies
- approval workflows for high-attention surface expansion
